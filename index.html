Commandes GitLab Essentielles

Créer un commit :

Enregistre les modifications.

git commit
Créer un commit avec un message :

Enregistre les modifications et ajoute un message descriptif pour ce commit.

git commit -m "Votre message ici"
Afficher les branches disponibles :

Liste toutes les branches disponibles dans le dépôt.

git branch
Supprimer une branche :

Supprime une branche localement.

git branch -d nom-branche
Renommer une branche :

Renomme la branche actuelle.

git branch -m nouveau-nom
Changer de branche :

Permet de passer d'une branche à une autre.

git checkout nom-branche
Créer et basculer vers une nouvelle branche :

Crée une nouvelle branche et bascule dessus en même temps.

git checkout -b nom-branche
Fusionner deux branches :

Fusionne la branche spécifiée avec la branche actuelle.

git merge nom-branche
Afficher le statut des fichiers :

Vérifie l'état des fichiers dans le dépôt local.

git status
Pousser les modifications :

Envoie les modifications locales vers le dépôt distant.

git push
Récupérer les modifications du dépôt distant :

Synchronise la branche locale avec les changements distants.

git pull
Initialiser un dépôt Git :

Crée un dépôt Git dans le répertoire courant.

git init
Cloner un dépôt :

Copie un dépôt distant dans un répertoire local.

git clone URL
Ajouter un fichier au suivi :

Ajoute un fichier spécifique à l'index pour le prochain commit.

git add fichier
Ajouter tous les fichiers au suivi :

Ajoute tous les fichiers modifiés à l'index pour le prochain commit.

git add .
Voir l'historique des commandes :

Affiche l'historique des commandes exécutées dans le terminal.

history
Changer de répertoire :

Permet de naviguer vers un autre répertoire.

cd chemin
Remonter d'un répertoire :

Retourne au répertoire parent.

cd ..
Lister les fichiers et répertoires :

Affiche la liste des fichiers dans le répertoire courant.

ls
Afficher le chemin courant :

Montre le chemin absolu du répertoire actuel.

pwd
Afficher tous les fichiers, y compris les fichiers cachés :

Affiche tous les fichiers, y compris les fichiers et dossiers cachés (commençant par un point).

ls -a
Pousser une branche et configurer le suivi :

Pousse une nouvelle branche et la lie à la branche distante correspondante.

git push --set-upstream origin nom-branche
